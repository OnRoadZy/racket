21.1 运行racket和gracket

可执行的gracket是和racket一样的，但是与一个控制台应用程序相比，作为一个GUI（图形用户界面）应用程序的表现上有小的调整。例如，gracket默认运行在GUI窗口交互方式代替控制台提示符。然而，GUI应用程序可以以普通的racket方式运行。

根据命令行参数、racket或gracket运行在交互模式（interactive mode）、模块模式（module mode）或加载模式（load mode）下。

21.1.1 交互模式

When racket is run with no command-line arguments (other than confguration options, like -j), then it starts a REPL with a >  prompt: 当racket用没有命令行参数（比其它的配置选项，如-j）运行时，那么用一个> 格式开始一个REPL：

  Welcome to Racket v6.12.

  >

为了提高你的REPL经验，参见xrepl；为获取GNU Readline支持信息，参见readline。

为初始化REPL的环境，racket首先需要racket/init模块，它提供了racket的所有，并且为了显示结果还安装了pretty-print。最后，如果存在的话，在启动REPL之前racket通过(find-system-path 'init-file)加载文件报告。

如果任何命令行参数被提供（除了配置选项），添加-i或--repl以重启REPL。例如,

  racket -e '(display "hi\n")' -i

在启动时显示“hi”，但还是出现了一个REPL。

如果模块导入标志出现在-i/--repl之前，它们取消racket/init的自动导入。这种行为可以被用来用一个不同的语言初始化REPL环境。例如,

  racket -l racket/base -i

使用更小的初始语言（加载速度更快）启动REPL。请注意，大多数模块都没有提供Racket的基本语法，包括函数调用语法和require。例如,

  racket -l racket/date -i

产生一个REPL，它在每一表达中失败，因为racket/date只提供很少的函数，而不是#%top-interaction和#%app绑定，它需要求值在REPL里的顶层函数调用。

如果一个模块导入标志出现在-i/--repl之后而不是在它之前，那么模块在racket/init之后导入以增强初始环境。例如,

  racket -i -l racket/date

除了racket的导出，用可获得的racket/date开始一个有用的REPL。

21.1.2 模块模式

如果一个文件参数在任何命令行开关（除了其它配置选项）之前提供给racket，那么这个文件作为一个模块导入，没有REPL启动。例如,

  racket hello.rkt

导入"hello.rkt"模块，然后退出。在文件名、标志或其它文件名之后的任何参数都被保存为命令行参数，以便通过current-command-line-arguments被导入模块使用。

如果使用命令行标志，那么-u或--require-script标志可以被用于显式地导入一个文件作为模块。 -t或--require标志是相似的，除了额外的命令行标志由racket处理，而不是保存为导入的模块。例如,

  racket -t hello.rkt -t goodbye.rkt

导入"hello.rkt"模块，然后导入"goodbye.rkt"模块，再然后退出。

-l或--lib标志类似于-t/--require，但它使用一个lib模块路径而不是一个文件路径导入一个模块。例如,

  racket -l raco

同样不用参数运行可执行的raco，因为raco模块是可执行的的主模块。

注意，如果你想传递命令行标志给上面的raco，你需要用一个--保护这个标志，因此racket不会试图解析它们本身：

  racket -l raco -- --help

21.1.3 加载模式

-f或--load标志支持直接load一个文件中的顶层表达式，而不是一个模块文件中的表达式。这种求值就像开始一个REPL并直接键入表达式，除非结果不打印。例如,

  racket -f hi.rkts

load "hi.rkts"并退出。请注意，加载模式通常是一个坏主意，在给有LISP/Scheme经验的读者的一个说明中获取原因的讲解；使用模块模式通常更好。

-e或--eval标志接受一个表达式直接求值。不同于文件加载，该表达式的结果被打印，就像在一个REPL中。例如,

  racket -e '(current-seconds)'

打印自1970年1月1日以来的秒数。

为了文件加载和表达式求值，顶层环境以与交互模式（interactive mode）相同的方式创建：除非首先指定另一个模块，否则必须导入racket/init。例如,

  racket -l racket/base -e '(current-seconds)'

可能运行得更快，因为它使用较小的racket/base语言来初始化求值环境，而不是用 racket/init。
