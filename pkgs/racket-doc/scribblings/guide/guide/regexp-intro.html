9.1 写regexp模式

一个字符串或字节字符串（byte string）可以直接用作一个正则表达式（regexp）模式，也可以#rx形成字面上的正则表达式值。例如，#rx"abc"是一个基于正则表达式（regexp）值的字符串，并且#rx"abc"是一个基于正则表达式（regexp）值的字节字符串（byte string）。或者，一个字符串或字节字符串可以以#px做前缀，如在#px"abc"中一样，稍微扩展字符串中模式的语法。

在一个正则表达式（regexp）模式的大多数角色都是相匹配的文本字符串（text string）中出现的自己。因此，该模式#rx"abc"匹配在演替中的一个字符串中包含的字符a、b和c。其它角色扮演的元字符（metacharacters）和字符序列作为元序列（metasequences）。也就是说，它们指定的东西不是字面上的自我。例如，在模#rx"a.c"，字符a和c代表它们自己，但元字符（metacharacter）.可以匹配任何字符。因此，该模式#rx"a.c"在继承中匹配一个a、任意字符和c。

当我们想要在一个Racket字符串或正则表达式的文字里面的一个字面上的\，我们必须将它转义以便它出现在所有字符串中。Racket字符串使用\作为转义字符，所以我们结束了两个\：一个Racket字符串\转义为正则表达式\，然后转义为.。另一个在Racket字符串里面需要转义的是"。

If we needed to match the character . itself, we can escape it by precede it with a \. The character sequence \. is thus a metasequence, since it doesn’t match itself but rather just .. So, to match a, ., and c in succession, we use the regexp pattern #rx"a\\.c"; the double \ is an artifact of Racket strings, not the regexp pattern itself. 如果我们需要匹配字符.本身，我们可以在它前面加上一个\。字符序列\.结果就是一个元序列（metasequence），因为它不匹配它本身而只是.。所以，在继承里匹配a、.和c，我们使用正则表达式模#rx"a\\.c"。双\字符是一个Racket字符串神器，它不是正则表达式（regexp）模式自己的。

regexp-quote函数接受一个字符串或字节字符串并产生一个正则表达式（regexp）值。当你使用regexp构建模式以匹配多个字符串，因为一个模式在它可以被使用在一个匹配之前被编译成了一个正则表达式（regexp）值。这个pregexp函数就像regexp，但使用扩展语法。尽管当它们可读，作为带#rx或#px的字面形式的正则表达式值被编译一次。

regexp-quote函数接受任意的字符串并返回一个模式匹配原始字符串。特别是，在输入字符串中的字符，可以作为正则表达式元字符用一个反斜杠转义，所以只有它们自己使他们安全地匹配。

> (regexp-quote "cons")

"cons"

> (regexp-quote "list?")

"list\\?"

regexp-quote函数在从一个混合的正则表达式（regexp）字符串和字面的字符串构建一个完整的正则表达式（regexp）是有用的。
