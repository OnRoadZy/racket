6.1 创建接口

(part ("(lib scribblings/guide/guide.scrbl)" "classes")) in (part ("(lib scribblings/guide/guide.scrbl)" "top")) introduces 类、对象和接口.

syntax

(interface (super-interface-expr ...) name-clause ...)

 
name-clause = id
  | (id contract-expr)

生成一个接口。id必须是互不相同的。

在interface表达式被求值时，每个super-interface-expr被求值（按顺序）。每个super-interface-expr的结果必须是一个接口值，否则exn:fail:object exception is raised。通过super-interface-expr返回的接口是新接口的超级接口，它被新接口进行所有扩展。任何类实现新接口，也实现所有的超接口。

一个interface表达式的结果是一个接口，它包括所有指定的id，加上来自于超级接口的标识符。在超接口之间的重复标识符名字被忽略，但如果一个超级接口包含interface表达式中的id之一，exn:fail:object exception is raised。一个给定的id可以搭配一个相应的contract-expr。

如果没有提供super-interface-expr，那么结果作为结果的派生需求是微不足道的：任何实现该接口的类必须从object%派生。否则，结果接口的实现需求是来自于它的超级接口的最具体的需求。如果超级接口不一致地指定派生需求，exn:fail:object exception is raised。

Examples:
(define file-interface<%>
  (interface () open close read-byte write-byte))
(define directory-interface<%>
  (interface (file-interface<%>)
    [file-list (->m (listof (is-a?/c file-interface<%>)))]
    parent-directory))

syntax

(interface* (super-interface-expr ...)
            ([property-expr val-expr] ...)
  name-clause ...)
 
name-clause = id
  | (id contract-expr)

就像interface，但是也对接口联想到通过property-expr用相应的val-expr生成的结构类型属性。

每当作为结果的接口（或一个从它派生的子接口）通过class*表被一个类单显式地实现时，每个属性都将其值附加到一个由类实例实例化的结构类型。具体来说，属性被附加到具有零立即字段的结构类型中，该字段被扩展为类的实例生成内部结构类型（因此，如果结构类型属性的守护者没有任何字段的信息，则可以访问）。

Example:
(define i<%> (interface* () ([prop:custom-write
                              (lambda (obj port mode) (void))])
               method1 method2 method3))
