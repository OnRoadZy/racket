12.10 文件包含

 (require racket/include) package: base
The bindings documented in this section are provided by the racket/include and racket libraries, but not racket/base.

本节所记录的绑定是由racket/include和racket库提供的，但racket/base库不提供。

syntax

(include path-spec)

 
path-spec = string
  | (file string)
  | (lib string ...+)

文件中的内联语法通过在include表达式这个地方里的path-spec指定。

一个path-spec类似于一个对require的mod-path表子集，但它指定一个内容不需要是模块的一个文件。也就是说，string是指一个使用一个不依赖于平台的相对路径的文件，(file string)指一个使用特定于平台的符号的文件，而(lib string ...)指的是一个集合中的一个文件。

如果path-spec指定一个相对路径，并且如果该源是一个完整的路径字符串，则该路径相对于include表达式的源解析。如果源是一个不完整的路径字符串，并且如果不是#f，那么path-spec相对(current-load-relative-directory)解析，否则，也可能相对(current-directory)解析。

所包含的语法给出了include表达式的词法上下文，而包含的语法的源位置是指其实际的源。

syntax

(include-at/relative-to context source path-spec)

类似于include，除了context的词法上下文被用于包含的语法，并且一个相对的path-spec用某方面解析到source的源。否则context和source元素将被扩展丢弃。

syntax

(include/reader path-spec reader-expr)

和include同样地，除了被表达式reader-expr产生的程序是用于读取包含文件，而不是read-syntax。

reader-expr在转换器环境里的扩展时间求值。因为它作为read-syntax的一个替换提供服务，这个表达式的值应该是一个过程，它接受两个输入——一个代表源的字符串和一个输入端口——同时产生一个语法对象或eof。该过程将反复调用，直到生成eof为止。

过程返回的语法对象应该有源位置信息，但通常没有词法上下文；语法对象中的任何词法上下文都将被忽略。

syntax

(include-at/relative-to/reader context source path-spec reader-expr)

合并include-at/relative-to和include/reader。
